# Atlas Stock Mailer MVP

사용자가 보유 종목을 직접 입력하면, 지정한 시간에 포트폴리오 점검 리포트를 이메일로 보내는 로컬 MVP입니다.

## 현재 가능한 것

- 수신 이메일 등록
- 매일 발송 시간 등록
- 보유 종목 수기 입력
- 리포트 미리보기
- 테스트 이메일 발송
- 프로그램이 켜져 있는 동안 매일 예약 시간에 자동 발송

## 제거한 것

- 주식 앱 화면 캡처 업로드
- OCR 자동 인식

OCR은 앱 화면마다 인식 품질 차이가 커서 잘못된 종목이 등록될 수 있습니다. 이 MVP에서는 신뢰도를 위해 수기 입력 방식으로 단순화했습니다.

## 메일 발송 설정이란?

SMTP는 쉽게 말해 **메일을 실제로 보내기 위한 발송 계정 설정**입니다.

수신 이메일은 "누구에게 보낼지"이고, SMTP는 "어떤 계정으로 보낼지"입니다.

실제 서비스라면 사용자는 SMTP를 몰라도 됩니다. 서비스 운영자가 서버에 발송 계정을 한 번 설정해두고, 사용자는 수신 이메일만 입력하면 됩니다.

다만 이 MVP는 개인 PC에서 바로 실행하는 로컬 프로그램이라서, 실제 이메일을 보내려면 발송 계정 정보가 필요합니다. 그래서 화면에는 **고급 설정**으로 접어두었습니다.

SMTP 설정이 없으면 실제 이메일은 보내지 않고 `email_preview.html` 파일로 미리보기만 저장합니다.

## Gmail로 실제 발송하려면

Gmail은 일반 비밀번호가 아니라 Google 계정에서 발급한 **앱 비밀번호**를 사용해야 합니다.

고급 설정에 입력할 값:

- SMTP 서버: `smtp.gmail.com`
- SMTP 포트: `587`
- 보내는 이메일 계정: 본인 Gmail 주소
- 보내는 사람 주소: 본인 Gmail 주소
- 앱 비밀번호: Google 계정에서 발급한 앱 비밀번호

## 더 편하게 만들려면

사용자 입장에서 편하게 만들려면 다음 구조가 좋습니다.

1. 사용자는 수신 이메일과 보유 종목만 입력
2. 운영자는 서버에 발송 계정을 한 번만 설정
3. 운영자는 서버에 시세/뉴스 데이터 연동을 한 번만 설정
4. 앱은 예약 시간에 자동으로 리포트 생성 및 발송

즉, 지금 보이는 SMTP 설정은 최종 사용자용이라기보다 **개발/운영자용 설정**입니다.

## 시세/뉴스 API

뉴스나 시세 확인을 위해 사용자가 Alpha Vantage API 키를 필수로 알아야 하는 것은 아닙니다.

Alpha Vantage는 시세/뉴스 데이터를 가져오기 위한 여러 선택지 중 하나일 뿐입니다. 실제 제품에서는 운영자가 서버에 데이터 제공자 API 키를 설정해두고, 사용자는 보유 종목만 입력하면 됩니다.

현재 로컬 MVP에서는 사용자 화면에서 API 키 입력을 제거했습니다. 데이터 연동이 아직 설정되지 않은 경우 리포트에는 "시세/뉴스 데이터 연동이 아직 설정되지 않았습니다"라고 표시됩니다.
